---
title: PostHTML configuration
description: Configure PostHTML parser options and register custom plugins in Maizzle
---

Maizzle uses PostHTML for templating and transformations, and you can configure it or even register plugins that can further transform your HTML emails.

## Options

PostHTML is configured under `build.posthtml.options` in your `config.js`.

### directives

<ParamField path="directives" type="array" default="[]">
  You can configure the PostHTML parser to correctly process custom directives.

  For example, you may tell it to ignore `<?php ?>` tags instead of treating them as HTML:

  ```js config.js
  export default {
    posthtml: {
      options: {
        directives: [
          { name: '?php', start: '<', end: '>' },
        ]
      }
    }
  }
  ```
</ParamField>

### xmlMode

<ParamField path="xmlMode" type="boolean" default="false">
  Enable `xmlMode` if you're using Maizzle to output XML content, and not actual HTML.

  ```js config.js
  export default {
    posthtml: {
      options: {
        xmlMode: true,
      }
    }
  }
  ```
</ParamField>

### decodeEntities

<ParamField path="decodeEntities" type="boolean" default="false">
  Set this to `true` to have entities within the document decoded.

  ```js config.js
  export default {
    posthtml: {
      options: {
        decodeEntities: true,
      }
    }
  }
  ```
</ParamField>

### lowerCaseTags

<ParamField path="lowerCaseTags" type="boolean" default="false">
  Set this to `true` to output all tags in lowercase. Works only when `xmlMode` is disabled.

  ```js config.js
  export default {
    posthtml: {
      options: {
        lowerCaseTags: true,
      }
    }
  }
  ```
</ParamField>

### lowerCaseAttributeNames

<ParamField path="lowerCaseAttributeNames" type="boolean" default="false">
  Output all attribute names in lowercase.

  <Warning>This has a significant impact on speed.</Warning>

  ```js config.js
  export default {
    posthtml: {
      options: {
        lowerCaseAttributeNames: true,
      }
    }
  }
  ```
</ParamField>

### recognizeCDATA

<ParamField path="recognizeCDATA" type="boolean" default="false">
  Recognize CDATA sections as text even if the `xmlMode` option is disabled.

  <Note>If `xmlMode` is enabled, CDATA sections will always be recognized as text.</Note>

  ```js config.js
  export default {
    posthtml: {
      options: {
        recognizeCDATA: true,
      }
    }
  }
  ```
</ParamField>

### recognizeSelfClosing

<ParamField path="recognizeSelfClosing" type="boolean" default="true">
  If enabled, self-closing tags will trigger the `onclosetag` event even if `xmlMode` is disabled.

  <Note>When `xmlMode` is enabled self-closing tags will always be recognized.</Note>

  Maizzle sets this to `true` to ensure self-closing tags like those of Components are rendered correctly.

  ```js config.js
  export default {
    posthtml: {
      options: {
        recognizeSelfClosing: true,
      }
    }
  }
  ```
</ParamField>

### sourceLocations

<ParamField path="sourceLocations" type="boolean" default="false">
  If set to `true`, AST nodes will have a `location` property containing the `start` and `end` line and column position of the node.

  ```js config.js
  export default {
    posthtml: {
      options: {
        sourceLocations: true,
      }
    }
  }
  ```
</ParamField>

### recognizeNoValueAttribute

<ParamField path="recognizeNoValueAttribute" type="boolean" default="true">
  If set to `true`, PostHTML will render attributes with no values exactly as they were written and will not add `=""` to them.

  ```js config.js
  export default {
    posthtml: {
      options: {
        recognizeNoValueAttribute: true,
      }
    }
  }
  ```
</ParamField>

### singleTags

<ParamField path="singleTags" type="array" default="[]">
  Use the `singleTags` option to tell PostHTML to treat custom tags as self-closing.

  <Warning>This needs to be used in conjunction with `closingSingleTag` to tell PostHTML how to close the tag, otherwise you will end up with an unclosed tag.</Warning>

  ```js config.js
  export default {
    posthtml: {
      options: {
        singleTags: ['custom'],
        closingSingleTag: 'slash', // see docs below
      }
    }
  }
  ```

  You may then use the `<custom />` tag as self-closing:

  ```html emails/example.html
  <custom name="opencounter" type="tracking" />
  ```
</ParamField>

### closingSingleTag

<ParamField path="closingSingleTag" type="string" default="default">
  Define the closing format for single tags.

  By default it will not close self-closing tags that it knows about:

  ```html emails/example.html
  <img>
  <p></p>
  ```

  Available options:

  <Accordion title="tag">
    Will add a closing tag.

    ```js config.js
    export default {
      posthtml: {
        options: {
          singleTags: ['custom'],
          closingSingleTag: 'tag',
        }
      }
    }
    ```

    ```html emails/example.html
    <custom></custom>
    ```
  </Accordion>

  <Accordion title="slash">
    Will add a closing slash.

    ```js config.js
    export default {
      posthtml: {
        options: {
          singleTags: ['custom'],
          closingSingleTag: 'slash',
        }
      }
    }
    ```

    ```html emails/example.html
    <custom />
    ```
  </Accordion>
</ParamField>

### quoteAllAttributes

<ParamField path="quoteAllAttributes" type="boolean" default="true">
  Disable if you want to remove quotes on all attributes.

  ```js config.js
  export default {
    posthtml: {
      options: {
        quoteAllAttributes: false,
      }
    }
  }
  ```

  ```html emails/example.html
  <img src=example.jpg>
  ```
</ParamField>

### replaceQuote

<ParamField path="replaceQuote" type="boolean" default="true">
  Replaces quotes in attribute values with `&quote;`.

  ```js config.js
  export default {
    posthtml: {
      options: {
        replaceQuote: false,
      }
    }
  }
  ```

  ```html emails/example.html
  <!-- `true` (default) -->
  <img src="<?php echo $foo[&quote;bar&quote;] ?>">

  <!-- `false` -->
  <img src="<?php echo $foo["bar"] ?>">
  ```
</ParamField>

### quoteStyle

<ParamField path="quoteStyle" type="number" default="2">
  Specify the attribute value quotes style.

  ```js config.js
  export default {
    posthtml: {
      options: {
        quoteStyle: 1,
      }
    }
  }
  ```

  ```html emails/example.html
  <!-- `2` (double quotes, default) -->
  <img src="example.png" onload="testFunc("test")">

  <!-- `1` (single quotes) -->
  <img src='example.png' onload='testFunc("test")'>

  <!-- `0` (based on attribute value) -->
  <img src="example.png" onload='testFunc("test")'>
  ```
</ParamField>

## Plugins

<ParamField path="plugins" type="array" default="[]">
  Register any PostHTML plugins that you would like to use, in the `plugins` array.

  You may register plugins to run either before all other plugins, or after all other plugins, by using the `before` and `after` keys.

  ```js config.js
  import spaceless from 'posthtml-spaceless'

  export default {
    posthtml: {
      plugins: {
        before: [
          spaceless()
        ]
      }
    }
  }
  ```
</ParamField>

### Custom plugins

You may write your own PostHTML plugins, right in your Maizzle `config.js` file.

For example, here's a plugin that adds a random number to all `<img>` src URLs:

```js config.js
export default {
  posthtml: {
    plugins: {
      after: [
        (() => tree => {
          const process = node => {
            if (node.tag === 'img' && node.attrs?.src) {
              const randomNumber = Math.floor(Math.random() * 10 ** 16).toString().padStart(16, '0')
              node.attrs.src = node.attrs.src + `?v=${randomNumber}`
            }

            return node
          }

          return tree.walk(process)
        })()
      ]
    }
  }
}
```

<Note>Note that this is a naive example that doesn't take existing query strings into account.</Note>
